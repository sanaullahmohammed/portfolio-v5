# CLAUDE.md — portfolio-v5

## What this project is

Personal portfolio site for Mohammed Sanaullah (distributed systems / full-stack engineer). React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui. Deployed to GitHub Pages at `/portfolio-v5/`.

## Key conventions

### Path alias
`@/` maps to `src/`. Always use `@/` imports, never relative `../../`.

### Components
UI primitives live in `src/components/ui/` and are generated by shadcn/ui — do not hand-edit them unless fixing a genuine bug. Page-level logic belongs in `src/pages/`, shared layout in `src/components/layout/`.

### Styling
Use Tailwind utility classes. Custom design tokens are CSS variables in `src/index.css`. Colors follow the `hsl(var(--token))` pattern. Do not hard-code hex/rgb values in components.

### Project data
All project content (descriptions, stack, links) lives in `src/data/projects.ts`. The `slug` field drives the `/work/:slug` route — keep slugs kebab-case and stable (changing them breaks existing URLs).

### SEO
Every page should use the `<Seo>` component from `src/components/Seo.tsx`. Site-wide constants (SITE_URL, SITE_NAME) live in `src/lib/seo.ts`.

### Routing
React Router v6 with `<BrowserRouter>`. The Vite base is `/portfolio-v5/` in production — do not use absolute paths starting with `/` in `<Link>` or `navigate()` calls; React Router handles this via the basename.

## Build & dev

```bash
npm run dev        # dev server on port 8080
npm run build      # production build → dist/
npm run preview    # serve dist/ locally
npm run lint       # ESLint check
```

## Deployment

Push to `main` → GitHub Actions builds Vite → uploads `dist/` as artifact → deploys to GitHub Pages. See `.github/workflows/deploy.yml`.

## TypeScript strictness

The project uses relaxed TypeScript settings (`noImplicitAny: false`, `strictNullChecks: false`). Do not tighten these without a coordinated migration — it will break the existing codebase.

## What NOT to do

- Do not add backend/API code — this is a fully static site.
- Do not add new UI primitives manually; use `npx shadcn add <component>` instead.
- Do not change `vite.config.ts` base path without also updating `src/lib/seo.ts` SITE_URL.
